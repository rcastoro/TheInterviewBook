This is the web client for my interview app (thin client). The DAL layer lives on my IIS express server on my local machine. The DAL is written in .NET Core 3 and ASP.NET, using SQL's (localdb) to persist data. The web client was initially written in React (which i did based on a friends suggestion, and was completely new to me). I found as I was learning React, JQuery and React don't play nice together because they are both DOM manipulators. So... the app was rewritten in vanilla javascript and JQuery based on the interview application initial parameters.



The web-client is hosted on a public domain and the back-end ASP.NET API is hosted on my local machine but the URL is provided to the internet via NGROK, which is a tunneling service. It provides my locally hosted API with a publicly accessible URL for my web client to call.



The users below consist of users stored in SQL's localdb on my machine at home, committed via the Sign Up button above. When users sign up, the /api/InterviewBookUser POST verb is called, which persists the user to the localdb database. Then, the same api endpoint in GET verb mode is called to display. The Hockey Teams available in the Sign Up drop down are also pulled from NHL.COM's API, via the backened ASP .NET application and are accessible via the /api/InterviewBookNHL GET verb request. Validation on the forms is done via jquery as requested. Thank you for taking the time to view the interview book, I look forward to speaking with you!
